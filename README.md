## _🖥️&nbsp;&nbsp;기획한 웹 페이지 소개_

&nbsp;&nbsp;&nbsp;&nbsp;⇒&nbsp;&nbsp; 2030 세대의 라이프스타일을 반영한 가구 쇼핑 환경을 기반으로,
<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;감각적인 연출과 직관적인 UI, 몰입도 높은 디자인을 통해 브랜드 가치를 전달하는 이커머스를 기획하였습니다.

```txt
① 기능 구현에 앞서 Figma를 활용한 디자인 기획 및 전체 화면 구조, 요구사항, 사용자 흐름 설계

② 단순 상품 판매를 넘어 2KEA_STUDIO만의 시선으로 재해석한 감각적인 공간 경험과
   2030세대를 타겟한 디자인 중심의 가구 이커머스 제작

③ 사용자 흐름을 고려하여 메인 페이지를 중심으로 서비스가 자연스럽게 이어지도록 페이지 구상

④ 메인 페이지에서 가구 컨셉 디자인과 함께 광고 모달 및 인스타그램 모달로 이커머스 경험 구성
   헤더 로고 클릭 시 메인 이동, 상품 이미지 클릭 시 상세 페이지로 즉시 연결

⑤ 상단 메뉴바를 통한 카테고리별 상품 리스트 탐색 및 이미지 클릭 시 상세 페이지 연결

⑥ 상세 페이지에서 장바구니 담기와 바로 구매 기능을 함께 제공하여 사용자가 원하는 방식으로 구매 가능

⑦ 메뉴바를 통해 회원가입 및 로그인 페이지 이동, 잘못된 경로 접근 시 404 페이지로 이동

⑧ 회원가입, 로그인, 상품 구매 및 취소 등 주요 액션마다 모달과 페이지 이동으로 알맞은 피드백 제공

⑨ 전체 페이지 및 모달을 Desktop / Tablet / Mobile 화면에 맞게 반응형으로 구현

⑩ Emotion Styled Components 기반 Theme 시스템으로 컬러, 폰트, 미디어쿼리를 상수화하여 일관된 디자인 시스템 구축

⑪ Zustand를 활용한 로그인 상태 및 장바구니 전역 상태 관리로 페이지 이동 간 데이터 유지

⑫ 로컬 스토리지를 활용하여 최근 본 상품, 리뷰, 주문 데이터, 팝업 노출 여부 등 클라이언트 데이터 영구 저장

⑬ Git 브랜치 전략을 활용한 팀 협업으로 기능별 독립 개발 및 충돌 최소화

    ⇒  모든 팀원들과 함께 각각의 아이디어를 내고 조율함으로써 기획과 디자인, 구조설계, 기능구현을 순차적으로 진행
```

## _📑&nbsp;&nbsp;기술적 문제와 해결 과정 (Trouble Shooting)_

### 구현 과정에서 발생한 문제와 해결

- 마이페이지 정보 수정 후 초기화 문제
  - 문제점&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;사용자 정보를 수정해도 새로고침 시 기존 정보로 다시 초기화되는 문제 발생<br>
  - 원인&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;API에서 받아온 회원 정보를 수정 후 서버로 재요청하지 않아,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;새로고침 시 기존 데이터가 다시 렌더링되는 구조<br>

  - 해결방안&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;수정된 데이터를 localStorage에 저장하고,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;저장값이 있을 경우 우선 사용하도록 로직 분기<br>

  - 결과&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;새로고침 이후에도 수정된 정보가 유지되도록 개선

    <br>

- 로고 애니메이션 타이밍 동기화 문제
  - 문제점&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;로고가 올라가는 애니메이션이 끝나기 전 헤더에서 로고가 미리 등장하는 문제 발생<br>
  - 원인&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;각 컴포넌트의 개별적인 마운트 시점과 브라우저 렌더링 성능 차이로 인해<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;애니메이션 시작 기준점이 서로 달라지는 현상 발생<br>

  - 해결방안&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;전역 컨텍스트(Context)로 애니메이션 시작 상태를 통합 관리하고,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;useMemo를 활용해 불필요한 리렌더링을 방지하며 타이밍을 동기화함<br>

  - 결과&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;로고의 등장과 이동 애니메이션 타이밍이 일치하도록 정교하게 개선

<br>

- 제품 리스트 반응형 기준 불일치
  - 문제점&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;페이지마다 동이한 제품 리스트 UI가 해상도별로 다르게 보임<br>
  - 원인&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;공통으로 사용해야 하는 반응형 기준과, 페이지별 조건식과 스타일 적용 방식이 다름<br>

  - 해결방안&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;데스크탑, 태블릿, 모바일 기준을 다시 정리하고,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;브레이크포인트와 조건명을 공통 규칙으로 통일하여 전체 페이지에 적용<br>

  - 결과&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;해상도에 따라 깨지던 현상이 줄고, 제품 리스트 UI의 일관성과 유지보수성이 개선

<br>

- 404 페이지 레이아웃 균형 문제
  - 문제점&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;404 페이지 중앙 콘텐츠와 하단 푸터 간격 과다, 화면 전체 불균형<br>
  - 원인&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;flex 정렬, min-height, margin, 배경 요소가 함께 영향을 주어 코드상 중앙 정렬과 실제 화면 중심 차이 발생<br>

  - 해결방안&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;중앙 콘텐츠와 푸터 간격 별도 조정, 배경 효과 위치 수정하여 시각적 균형 재조정<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;브레이크포인트와 조건명을 공통 규칙으로 통일하여 전체 페이지에 적용<br>

  - 결과&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;404 페이지 완성도 향상, 단순한 에러 페이지도 프로젝트 톤에 맞게 자연스럽게 정리 가능

### 후에 기획 및 디자인 측면에서의 보완점

- 모바일 환경 구매 기능 이슈 분석 및 개선 방향
  - 현황&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;쇼핑카트 페이지의 구매 기능에서 PC 환경에서는 정상적으로 동작했으나,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;모바일 환경에서는 구매 버튼 클릭 시 모달이 노출되지 않는 문제가 발생

  - 분석&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;이벤트 처리 및 포탈 구조는 정상적으로 동작했으나,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;모바일 환경에서의 렌더링 또는 인터랙션 차이에 대한 원인을 파악하지 못함

  - 보완 계획&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;모바일 환경에서의 이벤트 처리 및 렌더링 구조를 추가적으로 분석하고,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;다양한 디바이스 테스트를 통해 문제 원인을 명확히 파악하여 개선 예정

<br>

- 회원가입 인증 및 보안 처리 개선 방향
  - 현황&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;회원가입 폼 입력, 유효성 검사, 아이디 중복확인 및 회원가입 후 자동 로그인까지 기본 흐름을 구현함<br>

  - 분석&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;실제 서비스와 비교했을 때 이메일 및 휴대폰 인증 절차가 없고,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;비밀번호 암호화 및 서버 단 검증 처리(bcrypt 등)가 고려되지 않았으며,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;인증 토큰(JWT 등) 기반의 사용자 인증 흐름에 대한 이해와 적용이 부족한 상태<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;또한 에러 발생 시 사용자에게 명확한 피드백을 제공하지 못하는 한계 존재

  - 보완 계획&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;이메일·휴대폰 인증 절차를 추가하여 사용자 검증을 강화하고,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;bcrypt 기반 비밀번호 암호화 및 서버 단 유효성 검증을 적용하며,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;JWT 기반 인증 구조를 이해하고 로그인/회원가입 흐름에 반영할 예정<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;추가적으로 에러 처리 로직을 개선하여 사용자 피드백을 명확히 전달하도록 개선 예정

<br>

- 기획 및 디자인 프로세스 개선 방향
  - 현황&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;초기 기획 단계에서 사용자 경험(UX)과 전체 화면 흐름을 설계함<br>

  - 분석&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;치밀한 사전 계산 부족으로 개발 중 잦은 디자인 수정이 발생하였고,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;후반부 리뷰 페이지 추가로 인한 일정 관리의 어려움이 존재함<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;또한 촉박한 일정으로 인해 반응형 UI의 세부 완성도가 기대치에 미치지 못한 상태<br>

  - 보완 계획&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;기획 단계에 더 많은 시간을 투자하여 요구사항 정의를 명확히 하고,<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;피그마(Figma)를 활용한 와이어프레임 및 프로토타입 설계를 선행할 예정<br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;이를 통해 디자인 수정 리스크를 줄이고 반응형 디테일 등 작업 효율성을 높일 계획

- 리뷰 시스템 데이터 신뢰성 및 처리 구조 개선 방향

  ① 데이터 신뢰성 및 권한 관리<br>
  - 현황&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;클라이언트(localStorage) 기반으로 리뷰 데이터를 관리하여 데이터 위변조 및 신뢰성 문제가 존재<br>
  - 보완 계획&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;실제 서비스에서는 서버 기반 데이터 관리 및 인증/인가 처리 필요<br>

  ② 상태 관리 및 렌더링 구조<br>
  - 현황&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;setTimeout 기반 상태 처리로 렌더링 흐름이 불안정한 구조<br>
  - 보완 계획&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;useEffect 의존성 및 상태 흐름 정리가 필요하며 예측 가능한 상태 관리 구조로 개선 필요<br>

  ③ 에러 처리 및 사용자 피드백<br>
  - 현황&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;API 호출 실패 시 사용자에게 명확한 피드백이 제공되지 않음<br>
  - 보완 계획&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; ⇒&nbsp;&nbsp;&nbsp;네트워크 오류 및 예외 상황에 대한 UI 안내 및 에러 핸들링 개선 필요<br>
